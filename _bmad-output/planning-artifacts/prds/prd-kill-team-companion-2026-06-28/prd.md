**一句话**：把 KT 对局内「资格判定 × 条件修正 × 效果叠加 × 骰子推导」这条本该人脑拼的结算流水线，做成一个**可计算、可审计、无后端**的 Web 状态机，单屏共用，随用随开。

## 1. Vision

KT Lite 的痛苦不在「不懂规则」，在「**同一结算点要叠多层规则**」：可见性 × 掩护 × 命令 × 跨来源 buff/debuff × 骰子时机——规则书把它们拆散成平铺文字，玩家得在脑里临时拼一条流水线，而人脑工作记忆有限、叠加顺序隐性、状态每回合漂移，必然算错或卡壳。

本助手把这条流水线**外化成一个结算引擎**：持续持有棋盘与单位状态、生效中的 effect 栈、规则数据；在每个结算点算出「能不能做 / 怎么修正 / 谁先叠 / 投几颗」，并把每一步留痕可回看。规则查询、掷骰辅助只是这个引擎的「面」。

差异化锚点：现存工具（Wahstructor / New Recruit 类建队器、TTS 沙盒）**无人拥有对局内的结算流水线**。这是本产品的根价值。

v1 聚焦对局内结算，首发 KT Lite 核心规则 + 三阵营（军团兵 / 死亡天使 / 瘟疫战士），静态 Web 部署在 GitHub Pages，无后端，桌面浏览器与平板可用，单屏共用 pass-and-play。

## 2. Target User

### 2.1 Primary Persona
**Winst 本人**——构建者即用户。KT 玩家，桌面战棋爱好者，受够了多来源效果叠加时的结算摩擦，想自己造一个「算得对、算得清、随手能用」的伴随工具。也作为面基时与对手共用的单屏裁判。

### 2.2 Jobs To Be Done
- 对局时**不用脑算**叠加，把结算交给引擎，自己只做战术决策。
- 算到争议时**能回看依据**，秒平争论。
- 面基时**单屏共用**，免得抢书抢纸。
- 新阵营/规则版本来了**不用改代码**，加数据包即可。

### 2.3 Non-Users (v1)
- 只玩 KT 完整版（非 Lite）核心规则全本的玩家——v1 仅覆盖 Lite（协助/戒备/重掷优先级等缺口见 §8 OQ-4）。
- 要远程联机对局的玩家——v1 不做实时同步。
- 三阵营之外的 kill team 玩家——v1 仅 3 阵营。
- 只用手机的玩家——v1 面向桌面浏览器 + 平板。

### 2.4 Key User Journeys

- **UJ-1. Winst 在一击射击里把叠加全交给引擎。**
  Winst 在平板上选定攻击方特工与目标，引擎先验合法性（可见？命令？掩体？射程？控制范围内无己方？），点确认后投骰（电子或手投录入），引擎按 10 步流水线算出命中→致伤→防护→减免→伤亡，每步可展开看依据。算完状态自动更新，effect 到期自动处理。

- **UJ-2. 双方对「这刀怎么算」有争议，Winst 回放日志。**
  结算日志留痕每一步（输入骰、修正来源、抵挡分配、减免），任意一步可点开回看，依据落规则原文/来源，争议秒平。

*UJ 轻量（hobby/solo）；面基共用为有意义 UX，已捕获。*

## 3. Glossary

下游工作流与读者须使用以下术语，FR/UJ/SM 中不得引入同义词。

- **特工 (operative)** — 一个模型单位，持 APL/移动/豁免/耐伤。
- **APL / AP / CP** — 行动点上限 / 行动点（行动货币）/ 指挥点（计谋货币）。
- **耐伤 (Wounds)** — 生命值；归零 → **残废 (incapacitated)** 移出。低于起始=**受伤**，低于一半=**受创**（移动 -2"，武器命中 -1）。
- **可见 (Visible/LOS)** — 头部到目标底座外缘无阻碍直线。
- **掩护 (Cover)** — 控制范围内有阻碍地形即得；2" 内有他特工则无。给一次**掩护豁免**（保留一枚普通成功防御骰）。
- **被遮挡 (Obscured)** — 飞蝇云等制造的遮挡，阻止有效目标（与掩护不同）。
- **控制范围 (Within engagement range)** — 1" 内且可见，对称。
- **命令 (Order)** — 交战 (Engaged) / 隐匿 (Concealed)。隐匿有掩护不可为目标、不可射击/冲锋。
- **就绪 / 待机 (Ready / Exhausted)** — 激活前后状态。
- **有效目标 (Valid target)** — 交战命令+可见；或 隐匿+可见+无掩护。
- **攻击骰 / 防御骰** — 武器攻击属性数量颗 D6 / 固定 3 颗 D6。
- **普通成功 / 关键成功 (normal / critical success)** — ≥命中属性 / 自然 6；自然 1 必败。
- **出击 (Strike) / 格挡 (Parry)** — 成功攻击骰造伤 / 抵消对手成功。
- **致命 (Lethal x+)** — ≥x 视为关键成功。**暴击** = 关键成功造的伤害。
- **关键穿刺 (Critical Piercing)** — 防御骰减 1，仅攻击方有关键成功时触发。
- **恼人韧性 (Disgustingly Resilient)** — 受≥3 普通伤害时 D6 减伤（FNP 类）。
- **关键词 (Keyword)** — 阵营/子阵营标识，驱动规则触发。
- **教条 / 战斗条令 (Doctrine)** — 死亡天使的毁灭/战术/突击。
- **混沌印记 (Mark of Chaos)** — 军团兵阵营子标识（恐虐/纳垢/色孽/奸奇/无分）。
- **战团战术 (Chapter Tactic)** — 死亡天使 8 选 2 持久被动。
- **计谋 (Ploy)** — 战略/交战计谋，耗 CP。
- **转折点 (Turning point)** — 一回合；战斗共 4 个。
- **激活 (Activation) / 反应 (Reaction)** — 选命令→行动≤APL→翻待机 / 全待机时免费 1AP 介入。
- **后撤 (Fall back)** — 2AP；该激活内禁转移/冲锋。
- **毒素指示物 (Poison marker)** — 瘟疫战士独有；持有者每次激活受 1 伤害。
- **结算引擎 (Resolution Engine)** — 本产品核心：持有状态+effect 栈+规则并据此计算的子系统。

## 4. Features

### 4.1 结算引擎核心
**Description:** 持有单位状态与生效 effect 栈，骰源无关地执行射击/近战结算流水线，编码三阵营机制与明示叠加规则。Realizes UJ-1。

**Functional Requirements:**

#### FR-1: 特工状态托管
系统持续持有每个特工当前状态：耐伤/受伤受创、命令、就绪/待机、生效中 effect 及剩余转折点、装备、阵营子标识（印记/战团战术）。回合推进时自动让 effect 到期。Realizes UJ-1。
**Consequences:**
- 特工受伤/受创阈值自动判定并施加对应修正。
- effect 到期在正确转折点自动移除，不留残余。

#### FR-2: 两层属性模型 + 效果栈
属性区分**基础值**（含持徽手类「非修正」调整）与**修正**；每条修正标注类型（加/减命中、加/减伤害、升级/降级成功、数量增减、免疫）与叠加性（可叠 / 同源唯一 / 不与受创叠 / 条件触发）。Realizes UJ-1, 正确性护栏。
**Consequences:**
- 基础值调整与后续修正正确叠加（如持徽手 APL）。
- 同源修正不重复施加。

#### FR-3: 骰源无关结算
结算接受**电子投骰**或**物理骰手动录入**两种来源，二者进入同一修正流水线，结果一致。Realizes UJ-1。
**Out of Scope:** 物理骰图像识别（v1 手动录入）。

#### FR-4: 射击结算流水线
按规则执行 10 步射击序列：前置检查（有效目标/无遮挡/控制范围内无己方）→ 掷攻击骰 → 成功保留（命中修正先作用于命中属性）→ 攻击骰重掷/升级（平衡/无休/撕裂/严重/致命/精准）→ 防御骰（穿刺减数）→ 防御成功+掩护豁免（集中禁用）→ 抵挡分配 → 造伤（毁灭即时）→ 总伤害减免（FNP 类，每枚攻击骰减伤上限）→ 扣耐伤+后效。每步可展开看依据。Realizes UJ-1。
**Consequences:**
- 修正按上述插入顺序施加，顺序错则结果错。
- 「同源升级不叠」（严重抑制惩戒/撕裂）等明示规则被强制。

#### FR-5: 近战结算流水线
按规则执行近战序列：目标选择（控制范围内敌方，对方反击）→ 各选近战武器 → 同时掷攻击骰各自保留 → 从攻击方轮流结算（每枚选出击或格挡）→ 出击造伤/格挡抵消（决斗家普通可挡关键；残暴只能关键挡）→ 震荡附加 → 减免→扣耐伤→后效（灵魂盛宴等）。Realizes UJ-1。

#### FR-6: 阵营机制编码
编码三阵营招牌机制并接到正确流水线步骤：
- 军团兵：五混沌印记被动（恐虐严重/纳垢恼人生命力/色孽移动/奸奇远程严重/无分无休）+ 关键计谋/装备。
- 死亡天使：8 战团战术选 2 + 战斗条令 + 关键计谋（钢铁光环忽略一次普通伤害等）。
- 瘟疫战士：毒素武器规则+毒素指示物时机（流程结束时获得）+ 恼人韧性 + 关键计谋。
**Consequences:**
- 毒素指示物在「流程结束时」获得，故同次攻击的「剧毒」加成不作用于当次。
- 钢铁光环每场 1 次、释放恶魔每场 1 次等使用次数被追踪。

#### FR-7: 效果叠加规则编码
强制执行 12 条明示叠加/判定规则：同类成功升级不叠、近战禁<6 升级为关键（偏移肩盾）、同源减伤每枚攻击骰上限 1、命中-1 不与受创叠、掩护豁免额外保留不与制高点叠、战团战术不自我叠加、关键穿刺条件触发、毁灭即时并行、过热每行动一次等。Realizes UJ-1, 正确性护栏。
**Consequences:**
- 引擎不会双倍施加同向修正。
- 条件不满足时修正不触发。

### 4.2 棋盘状态与资格判定
**Description:** 持有棋盘状态（特工位置、地形/掩护特征），据此计算可见/掩护/遮挡/射程/控制范围，判定有效目标资格。这是「现实棋盘映射」的数字面，喂给一击交互。Realizes UJ-1。

#### FR-8: 棋盘状态托管（自由坐标）
系统以**自由坐标**持有特工位置（含底座/朝向）与地形/掩护几何特征；位置由玩家手动录入/拖放（v1 无扫描/token 自动同步）。Realizes UJ-1。
**Consequences:**
- LOS/掩护/遮挡/射程/控制范围均在自由坐标几何上计算——本产品最重的正确性表面，实现细节落架构阶段。
- 几何计算遇歧义（如地形精确边界）→ 降级人工裁定（见护栏）。

#### FR-9: 可见/掩护/遮挡/射程/控制范围计算
据棋盘状态计算 LOS、掩护、是否被遮挡、武器射程、是否在控制范围。

#### FR-10: 有效目标资格判定
综合命令+可见+掩护+遮挡+射程+控制范围内无己方，判定目标是否为有效目标；不合法时说明缺哪条。Realizes UJ-1。

### 4.3 对局推进与交互
**Description:** push 式推进回合与激活，强制行动合法性，提供「攻击该目标」一击交互与先验拦截。Realizes UJ-1。

#### FR-11: 回合/激活状态机
驱动 4 转折点 ×（战略阶段 + 交战阶段），先手权、CP 发放、就绪/待机翻转、交替激活、反应、计谋使用次数。Realizes UJ-1。

#### FR-12: 行动合法性
强制行动规则与后限制：AP≤APL、同激活不重复同行动、后撤后禁转移/冲锋、隐匿或在控制范围内禁射击、近战需敌方在控制范围内、阿斯塔特双近战/双射击特规。Realizes UJ-1。

#### FR-13: 「攻击该目标」一击交互
点目标 → FR-10 资格判定 → 自动选射击或近战流水线（FR-4/FR-5）→ 出伤亡，全程不切屏。Realizes UJ-1。

#### FR-14: 先验合法性拦截
结算/行动前先验合法性与前置条件；不合法则拦截并解释缺什么，而非算后纠错。

#### FR-15: push 式推进
引擎主动提示当前该做什么（该谁激活、可选行动、待结算项），玩家确认；非玩家主动翻查。

### 4.4 信任与可审计
**Description:** 自动结算的采纳前提——每步可退、可查。Realizes UJ-2。

#### FR-16: 单步回滚
任意结算/回合动作可单步前进、暂停、撤销（会话内）。

#### FR-17: 结算日志/回放
每次结算留痕（输入骰、每步修正来源、抵挡分配、减免、结果），任意历史步骤可回看/审计。Realizes UJ-2。

### 4.5 单屏共用
**Description:** 单设备共用，双方全公开。Realizes UJ-1。

#### FR-18: 单屏共用（全公开）
一台设备共用，**双方信息全公开可见**，不做隐藏；仅提示当前主动玩家。Realizes UJ-1。
**Consequences:**
- 刷新/关闭即丢失进行中对局（已知取舍，见 D-20）。

#### ~~FR-19~~ （已移除 — v1 不做存档/持久化，见 D-20）

### 4.6 建队与规则数据
**Description:** 建队约束校验 + data-driven 规则数据 + 内联规则查询。Realizes 建队 JTBD。

#### FR-20: 建队与约束校验
按阵营规则建队，实时校验**结构性合法性**（**非点数**——Lite 无点数模型，D-30）：①特工须从阵营可用列表选；②子阵营选择器约束（混沌印记 5 选 1、战团战术 8 选 2）；③装备/物品限制（如「重武器 ×1」）。非法即报错。

#### FR-21: data-driven 规则数据
规则以 JSON 数据包形式加载，引擎逻辑与数据分离；加阵营/版本/勘误 = 加/改数据包，不改代码。

#### FR-22: 首发 3 阵营 + KT Lite
随应用发布军团兵 / 死亡天使 / 瘟疫战士 + KT Lite 核心规则**结构化计算数据**（数值 / profile / effect 描述符），源自 `docs/rules`（官方转 md，**本地留存不入公开仓库**）。**不含 GW 规则原文**（D-29）。Realizes 三阵营 JTBD。

#### FR-23: 规则查询
任意关键词/effect/武器可内联查询，显示**引擎参数化要点**（数值 / profile / effect 描述符 / 触发步骤），**不显示 GW 规则原文**（D-29，公开仓库不含版权文本）；可指向玩家本地留存的原文位置。查询嵌进结算流，不需离开界面。Realizes 规则查询 JTBD。

### 4.7 目标点、得分与胜负
**Description:** 「占领目标点」任务全流程：部署 → 目标点控制 → VP 得分 → 胜负判定。Realizes 游戏全流程 JTBD（「从开局到最终胜负」）。

#### FR-24: 部署阶段
双方在各自降落区**交替部署**建队阵容特工到棋盘；非法位置（出降落区 / 与他特工重叠）拦截并说明。

#### FR-25: 目标点与控制
棋盘放置**目标点标识**；引擎据特工与目标点控制范围判定各方控制数（按 Lite 规则：范围内友方特工数优势等）。

#### FR-26: VP 得分
每**转折点结束**按控制目标点数结算双方胜利点（VP），累加。

#### FR-27: 胜负判定
4 转折点结束，**VP 总高者胜**；含平局规则。

**Notes:** v1 仅「占领目标点」一种任务；战术计划 / 二次目标 / 其它任务类型 v1 不做（见 §5）。

## 5. Non-Goals (Explicit)
- **不做** AI 对手/陪练。
- **不做** 远程实时联机/双端同步。
- **不做** 存档/持久化（刷新即重置）。
- **不做** 隐藏信息（v1 单屏全公开）。
- **不做** 战役追踪（伤亡/晋升跨局进化）。
- **不做** 教学层/新手引导（边玩边学）。
- **不做** KT 完整版核心规则全本（v1 仅 Lite；缺口见 §8）。
- **不做** 版本切换/多规则集共存（v1 仅 KT Lite 单一规则集）。
- **不做** 复杂任务系统/战术计划/二次目标（v1 仅「占领目标点」一种任务）。
- **不做** 三阵营外的 kill team。
- **不做** 手机优先布局（v1 桌面浏览器+平板）。
- **不做** 官方 GW 数据授权/集成（hobby 自用，规则源自玩家自有官方转 md）。

## 6. MVP Scope

### 6.1 In Scope
- 结算引擎核心（FR-1..FR-7）：状态托管、两层属性+效果栈、骰源无关、射击/近战流水线、阵营机制、叠加规则。
- 棋盘状态与资格判定（FR-8..FR-10）。
- 对局推进与交互（FR-11..FR-15）：回合状态机、行动合法性、一击交互、先验拦截、push 推进。

### Cross-Cutting NFRs
- **规则正确性（头号）**：引擎对已编码交互的输出须与规则源一致；任何算错视为缺陷。靠 FR-7 叠加规则编码 + FR-16/17 可审计兜底。
